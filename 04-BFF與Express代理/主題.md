# 04 - BFF 與 Express 代理

## 主題說明

在前端開發時，我們常常需要串接第三方的服務或 API（例如：Google Gemini API、OpenAI 等）。如果我們將 API Key 直接寫在前端（React）程式碼中，這些機密資訊就會隨著網頁一起被下載到使用者的瀏覽器，帶來極大的**資安風險**。

此外，直接從前端呼叫第三方 API 時，也經常會遇到 **CORS（跨來源資源共用）**的阻擋問題。

**BFF (Backend-For-Frontend) 架構**正是為了解決這些問題而生！
本單元將帶領你，從無到有將一個單純的 Vite React 專案，擴充並加入一個專屬於前端的 **Express 代理伺服器（Proxy Server）**。
- **安全性**：將 API Key 安全地藏在後端。
- **解決跨域**：由後端代替前端去呼叫第三方服務，藉此避開瀏覽器的 CORS 限制。

---

### 學習目標

- 建立一個全新的 Vite (React + TS) 專案。
- 學會在專案內建立 Node.js + Express 基礎後端。
- 掌握環境變數 `.env` 的正確觀念（後端專用 vs 前端專用）。
- 理解 Vite 伺服器的 `proxy` 設定，解決本機開發時的 CORS 問題。
- 使用 `concurrently` 同時啟動前後端伺服器，並學習如何為上線部署做準備。
