# 03.5 - 單元測試（Unit Tests）：Vitest / Jest

## 主題說明

當你開始把 AI 生成的專案「改成你想要的樣子」時，最常遇到的痛點是：
改了 A，B 壞了；修了 B，C 又壞了。

**單元測試（Unit Tests）** 的目的就是把「你希望程式永遠成立的規則」寫成可重複執行的檢查，
讓你每次改動都能快速確認：核心行為沒有被破壞。

本章以 **Vitest / Jest** 為主，帶你在 **Node.js** 與 **React** 專案中建立可執行的單元測試。

---

## 學習目標

- 理解單元測試在專案中的角色（何時該寫／不該寫）
- 在既有專案加入測試工具、測試環境與 `npm run test`
- 撰寫最常見的兩種測試：
  - 純函式測試（不依賴 DOM）
  - React 元件測試（需要 DOM / JSDOM）
- 會使用最常見的 Mock：時間、亂數、網路請求（fetch）

---

## 工具選擇（快速建議）

| 你的專案類型 | 建議工具 | 原因 |
|---|---|---|
| Vite + React | Vitest | 與 Vite 整合最好、設定量小 |
| Next.js | Jest（或 Vitest） | Next.js 社群範例多；Vitest 也可用但需留意設定 |
| Node.js（純後端） | Vitest 或 Jest | 兩者皆可，依團隊習慣 |

> 本講義以 **Vitest** 當作預設示範（對 Vite / TS / ESM 友善），並在需要時補充 Jest 差異。

---

## 把單元測試加進你的專案（參考流程）

以下以「**既有的 Vite + React + TypeScript 專案**」為例（Next.js 也能參考相同概念）。

### 1) 安裝套件

在你的專案目錄下執行：

```bash
npm i -D vitest @vitest/ui jsdom
npm i -D @testing-library/react @testing-library/jest-dom
```

### 2) 加入測試 scripts

在你的 `package.json` 加上：

```json
{
  "scripts": {
    "test": "vitest",
    "test:ui": "vitest --ui",
    "test:run": "vitest run"
  }
}
```

### 3) 設定 Vitest（Vite 專案）

在 `vite.config.ts` 加上 `test` 區塊：

```ts
import { defineConfig } from "vite";

export default defineConfig({
  test: {
    environment: "jsdom",
    setupFiles: "./src/test/setup.ts"
  }
});
```

### 4) 建立測試初始化檔

新增 `src/test/setup.ts`：

```ts
import "@testing-library/jest-dom/vitest";
```

### 5) 寫第一個測試（純函式）

新增 `src/sum.ts`：

```ts
export function sum(a: number, b: number) {
  return a + b;
}
```

新增 `src/sum.test.ts`：

```ts
import { describe, expect, it } from "vitest";
import { sum } from "./sum";

describe("sum", () => {
  it("adds numbers", () => {
    expect(sum(1, 2)).toBe(3);
  });
});
```

執行：

```bash
npm run test
```
