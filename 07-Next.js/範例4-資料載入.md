# 範例 4：資料載入

## 目標

學會在 Server Component 中直接 `async` 載入資料，並理解外層 Server Component、內層靜態展示元件與 Client Component 的分工。

## 步驟 1：認識 Server Component

Next.js App Router 的元件**預設為 Server Component**，可：

- 直接使用 `async` 函式
- 在伺服器端執行，不暴露給客戶端
- 直接 `fetch` 或查詢資料庫

> 本章會用「Static Component（靜態展示元件）」這個教學用語，指的是沒有 `"use client"`、沒有 `useState`、沒有 `useEffect`、沒有瀏覽器事件的純顯示元件。它不是 Next.js 官方第三種元件類型，實際上仍然屬於 Server Component 的常見寫法。

---

## 步驟 2：建立有資料載入的頁面

建立 `src/app/products/page.tsx`：

```tsx
async function getProducts() {
  // 模擬 API 或資料庫查詢
  return [
    { id: 1, name: "商品 A", price: 100 },
    { id: 2, name: "商品 B", price: 200 },
  ];
}

export default async function Products() {
  const products = await getProducts();

  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">商品列表</h1>
      <ul className="mt-4 space-y-2">
        {products.map((p) => (
          <li key={p.id}>
            {p.name} - ${p.price}
          </li>
        ))}
      </ul>
    </main>
  );
}
```

## 步驟 3：拆成外層 Server Component、內層 Static Component

上面的範例可以再拆得更清楚：外層頁面負責讀資料，內層元件只負責顯示資料。

```tsx
type Product = {
  id: number;
  name: string;
  price: number;
};

async function getProducts(): Promise<Product[]> {
  // Server script：這裡可以改成呼叫後端服務或查詢資料庫
  return [
    { id: 1, name: "商品 A", price: 100 },
    { id: 2, name: "商品 B", price: 200 },
  ];
}

function ProductList({ products }: { products: Product[] }) {
  // Static Component：只負責顯示，不需要送出額外的 client script
  return (
    <ul className="mt-4 space-y-2">
      {products.map((p) => (
        <li key={p.id}>
          {p.name} - ${p.price}
        </li>
      ))}
    </ul>
  );
}

export default async function Products() {
  // Server Component：在伺服器端取得資料，再把結果交給內層元件顯示
  const products = await getProducts();

  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">商品列表</h1>
      <ProductList products={products} />
    </main>
  );
}
```

這個結構可以先記成：

- **外層 Server Component（伺服器元件）**：負責資料載入、權限檢查、讀取環境變數、查詢資料庫。
- **內層 Static Component（靜態展示元件）**：只接收 props（屬性資料）並輸出 HTML（超文字標記語言），不處理互動。
- **Client Component（客戶端元件）**：只有需要 `useState`、`useEffect`、`onClick`、瀏覽器 API（應用程式介面）時才加上 `"use client"`。

### 哪些是 server script？哪些是 client script？

- 沒有 `"use client"` 的 `page.tsx`、`layout.tsx`、一般元件，預設是 server script（伺服器端腳本）。
- `getProducts()`、資料庫查詢、使用 token（權杖）或環境變數的程式，應該留在 server script。
- 檔案最上方有 `"use client"` 的元件，會變成 client script（瀏覽器端腳本）。
- client script 可以處理互動，但不應直接放資料庫密碼、私密 token（權杖）或只應該在伺服器端執行的邏輯。

## 步驟 5：重點整理

- `export default async function`：頁面元件可為 async
- `await getProducts()`：在伺服器端執行，資料不會送到客戶端
- 無需 `useEffect`、`useState` 載入資料
- 外層 Server Component 負責讀資料，內層 Static Component 可以只負責顯示
- 只有需要瀏覽器互動時，才建立 `"use client"` 的 Client Component
