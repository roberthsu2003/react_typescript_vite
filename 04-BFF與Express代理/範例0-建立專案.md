# 範例 0：建立專案與安裝依賴

## 目標

一步一步從零開始建立 `bff-example` 專案。這是一個 Vite (React + TypeScript) 專案。
為了實作 BFF 架構，我們也會一次性將**前端所需**以及**後端(Express)所需**的套件安裝完畢。

---

## 步驟 1：建立 Vite 專案

在 `04-BFF與Express代理` 資料夾中開啟終端機，執行以下指令來建立基本的 React 專案：

```bash
cd 04-BFF與Express代理
npm create vite@latest bff-example -- --template react-ts
```

> **若已存在 bff-example 資料夾**：可先重新命名或刪除後再執行上述指令。

---

## 步驟 2：進入專案並安裝前端基礎套件

執行以下指令，切換到新建立的資料夾並安裝預設依賴：

```bash
cd bff-example
npm install
```

---

## 步驟 3：安裝 BFF 架構所需套件

接下來，我們要一口氣將後續範例中會用到的後端工具、API 套件等一併安裝起來。

### 1. 安裝核心套件（正式環境所需）
這些套件會在我們的應用程式上線時被使用到，包含 Express 伺服器、跨域套件以及 Google Gemini SDK：

```bash
npm install express cors dotenv @google/genai concurrently
```
* **`express`**: 輕量級的 Node.js 網頁伺服器框架。
* **`cors`**: 處理跨域請求的 Express 中介軟體。
* **`dotenv`**: 幫我們從 `.env` 檔案中讀取環境變數（機密資訊）。
* **`@google/genai`**: 官方 Google Gemini API Node.js 客戶端。
* **`concurrently`**: 一個實用工具，讓我們可以「同時」啟動前端跟後端的開發伺服器。

### 2. 安裝開發用型別套件與工具（僅開發環境）
因為我們使用的是 TypeScript，所以必須安裝對應的型別定義檔，以及讓我們能直接執行 TypeScript 檔案的工具：

```bash
npm install -D @types/express @types/cors tsx
```

> **注意**：`-D` 代表安裝到 `devDependencies`，這些工具只在我們開發階段輔助使用，上線產品不需要它們。

---

## 步驟 4：建立後端伺服器資料夾

在 `bff-example` 專案的**根目錄**下，建立一個名為 `server` 的資料夾，這將是我們撰寫 Express 程式碼的地方。

```
bff-example/
├── server/       ← 建立此資料夾
├── src/
├── package.json
└── vite.config.ts
```

---

## 步驟 5：完成準備

到這裡，專案的基礎骨架以及所需的套件都已經準備妥當了！
你的 `package.json` 裡面應該已經可以看到剛剛安裝的所有套件。

不需要急著啟動 `npm run dev`，因為我們的前端與後端都還需要經過設定。
