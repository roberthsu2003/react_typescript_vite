## 步驟 1：單一 state 物件

多欄位時，與其多個 `useState`，可合併成一個物件（或 `useReducer`，進階）：

```tsx
import { useState, type ChangeEvent } from 'react'

type Form = {
  email: string
  password: string
}

export function LoginForm() {
  const [form, setForm] = useState<Form>({ email: '', password: '' })

  const update =
    (key: keyof Form) =>
    (e: ChangeEvent<HTMLInputElement>) => {
      setForm((f) => ({ ...f, [key]: e.target.value }))
    }

  return (
    <form
      onSubmit={(e) => {
        e.preventDefault()
        console.log(form)
      }}
    >
      <input value={form.email} onChange={update('email')} autoComplete="email" />
      <input value={form.password} onChange={update('password')} type="password" autoComplete="current-password" />
      <button type="submit">送出</button>
    </form>
  )
}
```

- **展開運算子** `{ ...f, [key]: value }` 做不可變更新。  
- `keyof Form` 限制只能更新 `email` / `password`。

## 步驟 2：驗證錯誤狀態

```tsx
type Errors = Partial<Record<keyof Form, string>>

function validate(f: Form): Errors {
  const e: Errors = {}
  if (!f.email.includes('@')) e.email = '請輸入有效 Email'
  if (f.password.length < 6) e.password = '密碼至少 6 字元'
  return e
}
```

提交前 `setErrors(validate(form))`，若有錯誤則不 `fetch`。

---

## 步驟 3：非受控元件（簡介）

使用 `ref` 讀 DOM 值（`defaultValue`）稱為**非受控元件**，適合簡單表單或與第三方套件整合；**多數 React 表單**仍以受控為主，狀態與畫面一致。

---

## 重點整理

| 概念 | 說明 |
|------|------|
| 受控 | `value` + `onChange` 由 React 狀態驅動 |
| 不可變更新 | `setForm(f => ({ ...f, key: v }))` |
| 驗證 | 提交前檢查，錯誤存於 `errors` 狀態 |
