## 步驟 1：`Partial<T>` — 全部屬性變為選用

`Partial<T>` 將 `T` 的**所有屬性**都變成 `?`（可選），常見於表單的「局部更新」場景：

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

// 全部屬性變選用：
type PartialUser = Partial<User>;
// 等同於：{ id?: number; name?: string; email?: string }

function updateUser(id: number, changes: Partial<User>): void {
  console.log(`更新 ${id}：`, changes);
}

updateUser(1, { name: "小明" });           // 只傳部分屬性，OK
updateUser(2, { name: "小華", email: "x@x.com" });
```

> **React 常見用法**：`setState` 接受 `Partial<State>`，讓呼叫端只傳需要更新的欄位。

---

## 步驟 2：`Required<T>` — 全部屬性變為必填

`Required<T>` 與 `Partial` 相反，把所有選用屬性（`?`）改為必填：

```typescript
interface Config {
  host?: string;
  port?: number;
}

type StrictConfig = Required<Config>;
// 等同於：{ host: string; port: number }

const cfg: StrictConfig = { host: "localhost", port: 3000 }; // 兩個都必須填
```

---

## 步驟 3：`Pick<T, K>` — 只保留指定屬性

從型別 `T` 中**挑選**出指定的屬性 `K`，形成新型別：

```typescript
interface Article {
  id: number;
  title: string;
  content: string;
  author: string;
  createdAt: Date;
}

// 列表頁只需要 id、title、author
type ArticlePreview = Pick<Article, "id" | "title" | "author">;

const preview: ArticlePreview = {
  id: 1,
  title: "TypeScript 入門",
  author: "小明",
};
console.log("preview:", preview.title);
```

---

## 步驟 4：`Omit<T, K>` — 移除指定屬性

`Omit<T, K>` 從型別 `T` 中**排除**屬性 `K`，保留其他：

```typescript
// 新增文章時，id 由資料庫產生，不需要傳
type NewArticle = Omit<Article, "id" | "createdAt">;

const draft: NewArticle = {
  title: "Vite 快速上手",
  content: "...",
  author: "小華",
};
console.log("draft:", draft.title);
```

> **Omit vs Pick**：  
> - 要保留的屬性**少**時用 `Pick`  
> - 要移除的屬性**少**時用 `Omit`

---

## 步驟 5：`as const` — 字面量推斷與唯讀

在物件或陣列後加上 `as const`，TypeScript 會將值**推斷為最精確的字面量型別**，並讓整個結構唯讀：

```typescript
// 沒有 as const → 推斷為 string[]
const colors1 = ["red", "green", "blue"];

// 加上 as const → 推斷為 readonly ["red", "green", "blue"]
const colors2 = ["red", "green", "blue"] as const;

// 常用：定義有限的設定值
const DIRECTIONS = {
  up: "UP",
  down: "DOWN",
  left: "LEFT",
  right: "RIGHT",
} as const;

type Direction = typeof DIRECTIONS[keyof typeof DIRECTIONS];
// Direction = "UP" | "DOWN" | "LEFT" | "RIGHT"

function move(dir: Direction): void {
  console.log("移動：", dir);
}
move(DIRECTIONS.up); // "UP"
```

## 步驟 6：型別斷言（`as`）

當你比 TypeScript 更了解某個值的型別時，用 `as` 明確告知編譯器：

```typescript
// 例：DOM 操作 — querySelector 回傳 Element | null
const input = document.querySelector("#name") as HTMLInputElement;
// 現在可以使用 .value
// input.value = "小明"

// 常見於 JSON 解析
const raw = JSON.parse('{"id":1}') as { id: number };
console.log("id:", raw.id);
```

> **注意**：`as` 是**告知**編譯器，不是執行期轉換。若實際值不符，執行時仍可能出錯。  
> 盡量只在「你確實知道型別」時使用。

---

## 步驟 7：非空斷言（`!`）

在可能為 `null | undefined` 的值後加 `!`，告訴 TypeScript「我確定這裡不是 null」：

```typescript
function getTitle(el: HTMLElement | null): string {
  // 若你確定 el 存在，可以用 !：
  return el!.title;
  // 等同於 (el as HTMLElement).title
}

// 更安全的寫法（推薦）：
function getTitleSafe(el: HTMLElement | null): string {
  if (!el) return "";
  return el.title;
}
```

> **建議**：優先用 `if` 或可選鏈（`?.`）排除 null，`!` 只在確定安全時使用。

## 步驟 8：`unknown` vs `any`

| | `any` | `unknown` |
|---|---|---|
| 可賦值給任何型別 | ✅ 是 | ❌ 否（需先檢查） |
| 可對其呼叫方法 | ✅ 是 | ❌ 否（需先縮窄） |
| 型別安全 | 低（放棄檢查） | 高（強制先檢查） |
| 適合場景 | 快速移植 JS、偶發情況 | API 回應、外部資料輸入 |

```typescript
function processAny(value: any) {
  console.log(value.toUpperCase()); // TypeScript 不會報錯，但執行期可能炸
}

function processUnknown(value: unknown) {
  // console.log(value.toUpperCase())  // ❌ 必須先縮窄
  if (typeof value === "string") {
    console.log(value.toUpperCase()); // ✅ 安全
  }
}

processUnknown("hello");  // HELLO
```

> **原則**：有型別不確定的情況，優先用 `unknown` 而非 `any`，讓 TypeScript 幫你抓潛在問題。

## 重點整理

| 工具型別 / 語法 | 用途 |
|---|---|
| `Partial<T>` | 全部屬性變選用，常用於局部更新 |
| `Required<T>` | 全部屬性變必填，去除所有 `?` |
| `Pick<T, K>` | 從 T 保留指定屬性 K |
| `Omit<T, K>` | 從 T 移除指定屬性 K |
| `as const` | 推斷為最精確字面量型別，使結構唯讀 |
| `value as Type` | 型別斷言：告知編譯器實際型別 |
| `value!` | 非空斷言：確保值不是 null/undefined |
| `unknown` | 型別安全的「不確定」，必須先檢查才能使用 |
| `any` | 跳過型別檢查，盡量避免 |
