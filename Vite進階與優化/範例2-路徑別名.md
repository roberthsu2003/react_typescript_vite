# 範例 2：路徑別名 (Aliases)

## 目標

解決深層資料夾結構帶來的「路徑地獄」問題，例如將 `import x from '../../../../components/x'` 簡化為 `import x from '@/components/x'`。

---

## 步驟 1：建立直覺的路徑觀念

在專案中，我們常以 `@` 代表 `src/` 這個根目錄。要達成這個效果，必須在 **Vite** 和 **TypeScript** 兩處分別進行設定。新手常因漏設一處而報錯。

---

## 步驟 2：讓 Vite 認得別名設定

開啟 `vite-advanced-demo/vite.config.ts`。

找到 `resolve.alias` 設定區塊：
```typescript
import path from 'path';

// ...
resolve: {
  alias: {
    '@': path.resolve(__dirname, 'src'),
  },
},
```
這告訴 Vite 在打包與伺服器運作時，看到 `@` 就等同於找到 `src/` 目錄。
*(註：專案需要安裝 `@types/node` 套件，才能在設定檔使用 `path` 模組)*

---

## 步驟 3：讓編輯器 (TypeScript) 認得別名

如果只設定 Vite，VSCode 的 TypeScript 會報錯說「找不到模組」。

請開啟 `vite-advanced-demo/tsconfig.app.json`。
找到 `compilerOptions` 內的 `paths` 設定：
```json
"compilerOptions": {
  "paths": {
    "@/*": ["./src/*"]
  }
}
```
這能讓編輯器正常提供自動補全，並且不再報錯。

---

## 步驟 4：實際驗證

開啟 `vite-advanced-demo/src/App.tsx`，看看程式最上方是不是有類似這樣的引用方式：
```typescript
import './App.css'
// 未來引用元件可以寫成： import SomeComponent from '@/components/SomeComponent';
```
透過 `@/`，不管這個檔案在哪個層級的資料夾，引用路徑都能保持乾淨簡潔。

---

## 完成

別名設定雖只是一次性的工作，但能大幅提升整個專案的開發體驗！
