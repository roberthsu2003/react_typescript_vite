
# 10 - Next.js × Vercel 雲端原生功能

## 主題說明

本單元以 **Next.js 16** 為框架，介紹 Vercel 平台的五項雲端原生功能。

這些功能代表了現代網頁開發從「自己管伺服器」轉向「雲端原生（Cloud Native）」的趨勢——  
把 Next.js 部署到 Vercel 後，複雜的後端問題（檔案儲存、快取、效能優化、邊緣運算）都已被平台原生解決。  
你只需要寫少量程式碼，即可獲得業界等級的基礎設施。

---

### 學習目標

- 理解 **Route Handlers** 如何自動成為 Serverless Functions，掌握 App Router 的 API 寫法
- 使用 **`<Image>`** 元件實現圖片自動優化（WebP 轉換、Lazy Load、防止版面位移）
- 了解 **Edge Runtime** 的使用時機、優勢與限制，與 Node.js Runtime 的差異
- 使用 **Vercel Blob** 儲存使用者上傳的檔案，取得公開 URL
- 使用 **Upstash Redis**（取代已停售的 Vercel KV）實作計數器與 Rate Limiting

### 練習專案建議（一次建立、全章共用）

本章五個範例建議都在同一個 Next.js 練習專案中完成（避免每個範例都重建專案、重設環境變數）。

> 範例 1 會帶你建立專案，你也可以先在開始前自行建立：

```bash
npx create-next-app@latest cloud-features --typescript --tailwind --app --src-dir --no-import-alias
cd cloud-features
```

- `npx create-next-app@latest cloud-features --typescript --tailwind --app --src-dir --no-import-alias`：在你目前所在的資料夾底下建立 `cloud-features/`，並初始化一個 Next.js 專案（含 TypeScript、Tailwind、App Router、`src/` 目錄結構）
- `cd cloud-features`：切換到新專案資料夾，後續才能執行 `npm install`、`npm run dev`

<details>
<summary>💡 指令與參數說明（可展開）</summary>

| 指令 / 參數 | 意思 |
|---|---|
| `npx` | 臨時下載並執行 CLI 工具（不需要全域安裝） |
| `create-next-app@latest` | Next.js 官方的「建立新專案」工具，使用最新版 |
| `cloud-features` | 新專案資料夾名稱（會產生 `cloud-features/`） |
| `--typescript` | 專案使用 TypeScript |
| `--tailwind` | 安裝並初始化 Tailwind CSS |
| `--app` | 使用 App Router（`app/`） |
| `--src-dir` | 原始碼放在 `src/` 目錄下（例如 `src/app/...`） |
| `--no-import-alias` | 不設定 `@/*` 等路徑別名（保持教材簡單） |

</details>
